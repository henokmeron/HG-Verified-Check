### Minimal fix (replace the `blocks` setup)

In **`VehicleReport (1).tsx`**, replace your current `blocks: Array<...> = [...]` with this dynamic builder:

```tsx
// Map schema keys → human titles (and doc names used by packageDocs)
const DOC_TITLES: Record<string,string> = {
  VehicleDetails: 'Vehicle Details',
  VehicleHistory: 'Vehicle History',
  DvlaTechnicalDetails: 'DVLA Technical Details',
  ModelDetails: 'Model Details',
  VehicleTaxDetails: 'Vehicle Tax Details',
  PncDetails: 'PNC Details',
  MiaftrDetails: 'MIAFTR Details',
  FinanceDetails: 'Finance Details',
  ValuationDetails: 'Valuation Details',
  SpecAndOptionsDetails: 'Spec & Options Details',
  BatteryDetails: 'Battery Details',
  TyreDetails: 'Tyre Details',
  VehicleImageDetails: 'Vehicle Images',
  MotHistoryDetails: 'Mot History Details',
};

const ORDER = [
  'VehicleDetails',
  'VehicleHistory',
  'DvlaTechnicalDetails',
  'ModelDetails',
  'VehicleTaxDetails',
  'PncDetails',
  'MiaftrDetails',
  'FinanceDetails',
  'ValuationDetails',
  'SpecAndOptionsDetails',
  'BatteryDetails',
  'TyreDetails',
  'VehicleImageDetails',
];

const blocks = ORDER
  .filter((k) => R?.[k] || P?.[k])
  .map((k) => ({
    title: DOC_TITLES[k] || k,
    docKey: DOC_TITLES[k] || k,
    schema: R?.[k],
    data: P?.[k],
    id: k.replace(/([a-z])([A-Z])/g, '$1-$2').toLowerCase(),
  }));
```

**Why this fixes it**

* Correct `docKey` per section → `SectionGate` hides/shows accurately for each package.
* Auto-includes all known top-level blocks → no missing sections when schema grows.
* Keeps your existing `SectionGate`/`Section` usage unchanged.
